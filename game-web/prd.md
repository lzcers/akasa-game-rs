## 1. 产品概述

### 1.1 产品定义
《幻世·人生回响》是一款基于 **AI Agent 模拟驱动** 的 Web 端非线性叙事游戏。产品采用 **模拟层与叙事层分离** 的架构：模拟层维护世界状态、因果逻辑与 NPC 自主行为；叙事层将状态变化实时渲染为优美连贯的文学文本。用户可自由定制人物设定与世界观，通过类似 GALGAME 的选择交互推动故事发展，最终走向必然收敛的结局，并获得一份个性化人生总结报告。

### 1.2 核心愿景
让每一位用户都能拥有一段独一无二的、由自己选择塑造的虚拟人生体验。故事并非被预先“写”死，而是在规则世界中真实“发生”——每一次选择都将引发蝴蝶效应，汇聚成一个逻辑自洽、不可预测的终局。

### 1.3 目标平台
- **主平台**：Web 浏览器（桌面端与移动端自适应）
- **技术栈**：React  + TypeScript 前端，Python (FastAPI) / Node.js 后端，LLM API 驱动叙事生成

## 2. 核心玩法与体验流程

### 2.1 开局定制：自定义人生原点

**2.1.1 人物塑造模块**
- **基础档案**：姓名、性别、年龄、外貌特征（关键词描述，AI 据此生成初始立绘与肖像文本）。
- **人格光谱**：三项核心特质滑动调节（勇气/谨慎、理性/感性、利己/利他），影响 AI Agent 行为倾向与选项权重。
- **人生烙印**：选择一个背景标签（如“背负诅咒的继承者”、“失忆的流浪剑客”），作为贯穿全剧的隐藏线索与结局收敛方向。

**2.1.2 世界观设定模块**
- **时代与舞台**：预设模板（蒸汽朋克、星际拓荒、东方玄幻）或自定义关键词。
- **世界记录**：定义世界的宏观冲突、秩序机制、禁忌与故事张力。

### 2.2 叙事进行：GALGAME 式交互与蝴蝶效应

**2.2.1 主界面布局**
- **中央舞台**：动态场景与人物立绘（AI 实时生成或风格化渲染）。
- **对话框**：叙事文本逐字流式输出，沉浸式阅读。
- **选项区域**：2-4 个模糊语义选项（如“说出真相，无论代价”），不暴露具体数值影响。

**2.2.2 蝴蝶效应机制**
- 玩家选择不直接跳转固定剧情分支，而是向 **世界状态数据库** 写入参数变更（如好感度、隐秘值、资源量、潜在事件标记）。
- 后续叙事由 AI 基于更新后的状态上下文生成，确保因果链的自然推演。
- **模拟层压力系统**：内置硬性倒计时（如“三十日后彗星降临”）与软性锁定阈值（核心变量触发专属事件链），保证故事在 1-2 小时游戏时长内必然收敛至结局。

### 2.3 落幕：个性化人生总结

结局触发后，生成《此生回响录》，包含：
- **生平纪事**：一篇文学性人物传记（500-800 字）。
- **关键转折点复盘**：列出 3 个蝴蝶效应的具体因果案例。
- **精神遗产评估**：基于主角行为产生的世界影响评价。
- **视觉纪念**：由 AI 生成的终幕画像与三张关键场景印象绘（CG）。

### 2.4 增强玩法机制

在基础交互之上，引入轻量系统提升策略感与可玩性。

| 功能模块 | 说明 | 体验目标 |
| :--- | :--- | :--- |
| **执念资源系统** | 每局赋予 3-5 点“执念”，可在关键选项倾注，强行提升选择成功率，但可能引发反噬。 | 从被动选择到主动博弈，增加高风险决策爽感。 |
| **命运窥镜** | 消耗“直觉值”预览某选项下一段文本的模糊片段（限 30 字）。 | 提供有限信息优势，制造悬念与猜疑乐趣。 |
| **世界切片新闻** | 界面偶现短时世界消息（如“远方城邦燃起烽火”），由模拟层全局变量触发。 | 暗示世界自动运转，增强世界辽阔感与错失感。 |
| **NPC 记忆介入** | 重要 NPC 会依据玩家过往行为主动介入叙事（信件、偶遇），对话引用具体事件。 | 强化 AI Agent 的鲜活度与玩家行为后果的严肃性。 |

## 3. 系统架构设计

### 3.1 模拟层

**职责**：维护世界状态、执行因果律、驱动 NPC Agent 自主行为，为叙事提供事实依据。

| 组件 | 功能 | 技术实现思路 |
| :--- | :--- | :--- |
| **世界状态机** | 存储 NPC 属性、全球变量、事件日志等结构化数据。 | PostgreSQL JSONB 字段，支持快照与回溯。 |
| **因果逻辑引擎** | 解析玩家选择，计算变量增减与事件触发条件。 | 规则脚本 + 轻量 LLM 辅助模糊推理。 |
| **AI Agent 循环** | 关键 NPC 拥有简易目标与记忆，玩家回合间自主更新状态。 | 基于 LLM 的简易 Agent 框架，低频率更新。 |

### 3.2 叙事层

**职责**：将模拟层的状态变化转化为符合世界观风格的优美文学文本。

| 组件 | 功能 | 技术实现思路 |
| :--- | :--- | :--- |
| **上下文组装器** | 收集当前环境、人物、关系、近期事件等结构化信息。 | 构建高密度 Prompt 上下文（世界卡、人物卡、历史摘要）。 |
| **文学渲染模型** | 将上下文生成为叙事文本。 | 调用 LLM API（如 Claude、GPT），思维链分步生成。 |
| **风格一致性控制** | 确保文本风格贴合设定（武侠、克苏鲁等）。 | Prompt 注入风格指令与 Few-shot 示例。 |
| **幻觉拦截器** | 校验生成内容是否违背模拟层事实（如角色已死）。 | 规则匹配 + NLI 模型实时校验，必要时修正或重生成。 |

### 3.3 交互时序流程

1. 用户点击选项 → 前端发送意图 ID。
2. 模拟层更新状态 → 触发 NPC Agent 反应 → 计算因果链。
3. 叙事层组装上下文 → LLM 流式生成后果文本与世界侧写。
4. 前端渲染新文本、更新立绘与场景，展示新选项。
5. 后台异步更新 NPC 潜在行为逻辑，准备下一轮。

## 4. 用户界面与交互设计

### 4.1 首页大厅
- 宽幅动态背景画卷，中央“开启新人生”按钮。
- 侧边“我的回廊”展示过往结局卡片，支持点击阅读《此生回响录》并生成分享卡片。

### 4.2 创角工坊
- 分步式卡片引导，包含人物与世界观设定。
- 提供“随机骰子”一键生成戏剧性组合。

### 4.3 叙事主舞台
- 自适应 AVG 布局，支持深色磨砂玻璃质感。
- 快捷功能栏：存档/读档（带剧情缩略图）、历史记录回看、人物关系图谱弹窗。
- 移动端适配：选项以大按钮置于底部，便于拇指操作。

### 4.4 结局展示页
- 沉浸式滚动阅读《此生回响录》，背景音乐舒缓。
- 交互彩蛋：点击总结中关键段落，触发“如果当时……”假想闪回文本。
- 蝴蝶效应因果涟漪图：交互式图表展示决策链与结局的关联。

### 4.5 智能分享模块
- 存档时自动生成精美分享卡片（含剧情摘要、角色引言、当前场景印象绘）。
- 结局后可一键生成包含三张 CG 印象绘的社交分享图。

## 5. 技术实现方案

| 层级 | 推荐技术                               | 说明 |
| :-------- | :--------------------------------- | :----------------------- |
| **前端** | React + TypeScript + Tailwind CSS  | 构建强交互 SPA，自定义 AVG 动效。 |

| **数据库** | PostgreSQL (主数据) + Redis (会话缓存)    | 支持 JSONB 存储世界状态快照。 |
| **图像生成** | Stable Diffusion API (如 Replicate) | 按需生成立绘、场景、结局印象绘。 |
| **实时通信** | WebSocket                          | 流式传输 LLM 生成文本，消除等待焦虑。 |

## 6. 产品亮点与差异化

1. **真正的非线性叙事**：摒弃剧情树，基于状态模拟实现因果推演，每一次游玩都是对初始变量的重新解算。
2. **会呼吸的 NPC**：AI Agent 具备记忆与主动性，世界在玩家“离线”时依然演化。
3. **叙事即艺术**：剥离数值面板，用户只面对纯粹的文学文本，沉浸感极强。
4. **强反馈闭环**：结局的个性化人生总结与可视化因果图提供极高情绪价值，激发分享与重玩。
5. **Web 端零门槛**：无需下载，即开即玩，便于传播。

## 7. 功能扩展路线图

为确保产品有序迭代，功能规划划分为四个阶段，各阶段目标明确、相互支撑，最终构建完整的长线生态。

### 7.1 第一阶段：核心闭环（P0 基础体验）

**目标**：交付可玩的最小可行产品（MVP），验证核心叙事循环与模拟层架构，提供完整的故事起始至结局体验。

| 编号 | 功能名称 | 详细描述 | 用户价值 | 优先级 |
| :--- | :--- | :--- | :--- | :--- |
| P0-1 | 基础角色与世界定制 | 实现人物档案、人格光谱、人生烙印三项设定；世界观模板选择与核心矛盾定义。 | 让玩家创造属于自己的故事原点。 | 必须 |
| P0-2 | 模拟层状态机与因果引擎 | 搭建世界状态数据库与规则脚本引擎，支持玩家选择触发变量变更与事件标记。 | 为非线性叙事提供底层因果支撑。 | 必须 |
| P0-3 | 叙事层流式生成 | 接入 LLM API，实现上下文组装与文本逐字流式输出。 | 提供沉浸式阅读体验，消除等待焦虑。 | 必须 |
| P0-4 | 硬性倒计时与软性锁定结局 | 实现压力系统，确保故事在 1-2 小时内收敛至结局。 | 保证每局游戏都有明确的终点与完整性。 | 必须 |
| P0-5 | 基础结局总结报告 | 生成一篇 500-800 字的生平纪事文本。 | 提供情感落点，完成体验闭环。 | 必须 |
| P0-6 | 存档/读档与历史记录 | 支持多档位存档（带剧情缩略图与摘要），可回看已读文本。 | 允许中断与回溯，降低试错成本。 | 必须 |
| P0-7 | 智能存档分享卡片 | 存档时自动生成包含剧情摘要与引言的精美图片卡片。 | 为 Web 端社交传播提供内容载体。 | 必须 |

**第一阶段验收标准**：
- 用户可完成一次从创角到结局的完整游戏流程。
- 故事内容具备基本因果连贯性，无明显逻辑断裂。
- 流式文本生成延迟低于 2 秒初始响应，后续 token 稳定输出。
- 结局报告文学质量达到可读水平。

### 7.2 第二阶段：玩法增强（P1 趣味提升）

**目标**：在核心闭环基础上引入策略层与可视化反馈，大幅提升游戏可玩性、重玩意愿与情绪高潮体验。

| 编号   | 功能名称       | 详细描述                                                                              | 用户价值                               | 优先级 |
| :--- | :--------- | :-------------------------------------------------------------------------------- | :--------------------------------- | :-- |
| P1-1 | 执念资源系统     | 每局赋予 3-5 点“执念”，关键选项处可倾注以提高成功概率，但伴随潜在反噬（如后续选项减少、触发负面事件）。界面需直观显示剩余执念点数。             | 将被动选择变为主动博弈，增加决策的策略深度与风险爽感。        | 高   |
| P1-2 | 命运窥镜       | 消耗“直觉值”（可通过特定选择或执念转化获取）预览某一选项下一段文本的模糊片段（约 30 字）。需设计预览文本的模糊化呈现方式（如毛玻璃效果）。          | 满足对另一条路的好奇心，制造悬疑张力，丰富交互层次。         | 高   |
| P1-3 | 世界切片新闻     | 在叙事间隙（每 3-5 次选择后）界面角落浮现短时世界消息，内容由模拟层全局变量（如经济指数、阵营声望）触发。消息停留 5 秒后自动消失。             | 暗示世界自动运转，增强宏观世界感与“错失”体验。           | 高   |
| P1-4 | NPC 记忆介入   | 关键 NPC 记录玩家过往交互行为。若长时间未互动，NPC 可能通过信件、偶遇等方式主动介入，对话中引用具体历史事件。需为每个重要 NPC 维护一个简易记忆向量。 | 让 AI Agent 显得有脾气、有记忆，强化玩家行为的后果重量感。 | 高   |
| P1-5 | 蝴蝶效应因果涟漪图  | 结局报告页面增加交互式图表，点击生平纪事中的关键句，图表高亮显示导致该结果的上游选择链。图表以力导向图或桑基图形式呈现。                      | 提供侦探式复盘快感，让玩家理解“为何如此”，激发重玩欲望。      | 高   |
| P1-6 | “如果当时”假想闪回 | 在因果涟漪图上为关键节点设置“闪回”按钮。点击后调用快照数据，LLM 生成一小段假想叙事（约 100 字），描述若选择另一分支可能的发展方向。           | 在结局时刻释放未竟心愿，提升情绪后劲与分享冲动。           | 高   |
| P1-7 | 结局三张印象绘 CG | 结局生成《此生回响录》时，调用 AI 绘图 API 生成三张关键场景图（如初遇、背叛、和解），插入报告中作为视觉高潮。                       | 提供独一无二的专属剧情 CG，极大增强结局收藏价值与社交分享动力。  | 高   |

**第二阶段验收标准**：
- 执念与直觉值系统界面友好，逻辑无漏洞。
- 世界切片新闻与 NPC 记忆介入自然融入叙事，不显突兀。
- 因果涟漪图能准确反映至少 80% 的主要因果路径。
- 假想闪回文本与主线叙事风格一致，无明显脱节。
- 三张印象绘 CG 风格统一，符合世界观设定，生成成功率 > 95%。

### 7.3 第三阶段：生态构建（P2 长线活跃）

**目标**：通过 UGC 工具、社交挑战与跨周目彩蛋，构建玩家社群生态，延长产品生命周期，形成自传播。

| 编号 | 功能名称 | 详细描述 | 用户价值 | 优先级 |
| :--- | :--- | :--- | :--- | :--- |
| P2-1 | 世界种子系统 | 每局开局生成唯一种子码（如 `E8F2-9A1C`）。玩家可输入他人分享的种子码，在完全相同初始条件下重玩。种子需编码所有创角设定与随机初始变量。 | 为社群挑战提供标准化基础，便于玩家交流与竞速。 | 中 |
| P2-2 | 挑战词条模式 | 在创角或载入种子时，可选择附加挑战词条（如“一生不许说谎”、“必须三十日内花光遗产”）。词条将强制约束玩家行为或改变模拟层规则。完成挑战可获得专属结局标签。 | 增加重玩的目标感与难度梯度，满足硬核玩家需求。 | 中 |
| P2-3 | 跨周目传承彩蛋 | 玩家完成一个结局后，下一局全新故事中，有较小概率（约 5%-10%）以彩蛋形式出现上一个人生的遗物或逸闻。例如，上局发明的机械鸟设计图变为下局武侠世界中的机关术残卷。彩蛋仅作为文本提及，不改变核心剧情走向。 | 制造“所有人生在同一精神宇宙中回响”的诗意连接，鼓励持续创作新故事。 | 中 |
| P2-4 | 创作者工坊（UGC 轻量版） | 开放“世界观规则卡”自定义功能：高级玩家可编辑 JSON 格式的规则模板，定义新的变量类型、事件触发条件与因果逻辑权重。通过审核后可上架创意工坊供他人订阅使用。 | 将产品从“讲故事的工具”升级为“创作平台”，激发社区创造力。 | 低（资源需求高） |
| P2-5 | 异步联机人生（实验性） | 在同一世界种子下，允许多名玩家创建不同角色（最多 3-5 人）。各自游玩时，其关键选择会以“世界消息”形式出现在他人游戏中，并轻微影响他人世界的变量（如某个 NPC 被另一玩家杀死则不可用）。采用异步机制，无需同时在线。 | 提供轻度社交耦合，让玩家感知到“他者”的存在，丰富故事可能性。 | 低（技术复杂度高） |
| P2-6 | 个人回廊扩展 | 我的回廊增加筛选、排序、收藏功能。支持将多个人生总结组合成“人生集”，并可生成合集封面分享。 | 提升内容管理体验，便于回顾与展示个人叙事历程。 | 中 |

**第三阶段验收标准**：
- 种子码能 100% 复现初始世界状态。
- 挑战词条逻辑正确执行，无绕过漏洞。
- 传承彩蛋触发概率稳定，内容自然不违和。
- UGC 工坊具备基础的内容审核与版本管理能力。
- 异步联机实验模式下世界状态同步延迟在可接受范围（分钟级）。

### 7.4 第四阶段：探索性功能（P3 长期愿景）

**目标**：探索技术与叙事的边界，引入前沿 AI 能力与沉浸式交互，定义下一代叙事游戏体验。

| 编号 | 功能名称 | 详细描述 | 用户价值 | 优先级 |
| :--- | :--- | :--- | :--- | :--- |
| P3-1 | 语音叙事与 AI 配音 | 结局报告与关键剧情段落支持 AI 语音朗读，音色与情绪由角色设定自动匹配。可选项：玩家自定义旁白音色。 | 丰富感官维度，提升沉浸感与无障碍体验。 | 低 |
| P3-2 | 实时动态立绘生成 | 角色立绘不再静态，而是根据当前情绪状态（文本分析得出）实时微调表情与姿态（通过 AI 生成局部变化图）。 | 让角色更加生动，增强情感传递。 | 低 |
| P3-3 | 多模态输入叙事 | 允许玩家在特定节点通过麦克风输入一句话，AI 解析意图后转化为游戏内选择。例如，面对 NPC 质问，玩家直接说出“我永远不会背叛你”。 | 提供前所未有的自由交互感，模糊玩家与角色的边界。 | 极低（技术前沿） |
| P3-4 | 记忆回廊数字藏品化 | 与合规的数字藏品平台合作，允许玩家将特别珍贵的《此生回响录》（含三张 CG）铸造为限量 NFT，作为其虚拟人生的永久数字证明。 | 为深度玩家提供终极收藏价值与身份象征。 | 极低（需合规与市场成熟） |

**第四阶段说明**：此阶段功能依赖 AI 技术与市场环境的发展，不作为近期开发承诺，但作为产品长期愿景的技术预研方向。

### 7.5 路线图总览表

| 阶段 | 核心目标 | 预计周期（迭代） | 关键交付物 |
| :--- | :--- | :--- | :--- |
| **第一阶段** | 交付可玩 MVP，验证核心循环 | 3-4 个月 | 基础游戏流程、结局总结、分享卡片 |
| **第二阶段** | 玩法增强，提升趣味与重玩性 | 4-6 个月 | 执念系统、命运窥镜、因果图、假想闪回、印象绘 CG |
| **第三阶段** | 生态构建，长线活跃与 UGC | 6-12 个月 | 种子挑战、传承彩蛋、创作者工坊（轻量） |
| **第四阶段** | 前沿探索，定义下一代体验 | 长期预研 | 语音叙事、动态立绘、多模态输入 |

**迭代原则**：
- 每个阶段完成后进行用户测试与数据复盘，根据反馈调整下一阶段优先级。
- 第二阶段中的高优先级功能（P1-1 至 P1-7）可依据资源情况分批次上线，但建议在第二阶段周期内全部完成。
- 第三阶段功能可根据社区活跃度与用户需求选择性投入，UGC 工坊与异步联机需谨慎评估技术成本。

## 8. 总结

《幻世·人生回响》通过 **模拟层与叙事层的架构分离**，在 Web 端实现了兼具开放世界自由度与文学品质的非线性叙事体验。产品以 AI Agent 模拟为引擎，以蝴蝶效应为乐趣核心，以个性化人生总结为情感锚点，致力于让每位用户都能创造并珍藏一段属于自己的虚拟人生。

本文档完整描述了产品的核心设计、系统架构、界面交互、技术方案与详尽的功能扩展路线图。通过四阶段迭代规划，产品将稳步从核心闭环走向玩法增强、生态构建与前沿探索，逐步成长为具有长期生命力的叙事平台。文档将作为后续详细设计与敏捷开发的基础依据。
